# 修复线上搜索功能指南

## 🎯 目标
修复线上（生产环境）数据库的搜索功能。

## ⚠️ 重要说明

**`npm run post-deploy` 修复的是 `DATABASE_URL` 指向的数据库**

- 如果 `.env` 配置的是本地数据库 → 修复本地
- 如果 `.env` 配置的是线上数据库 → 修复线上

## 🚀 推荐方案：使用临时环境变量

### 方法 1: 一次性修复（最安全）

```bash
# 1. 获取线上数据库 URL（从 Vercel 环境变量）
# 格式: postgresql://user:password@host:5432/database

# 2. 临时设置并运行修复
DATABASE_URL="postgresql://线上数据库URL" npm run fix:prod
```

**优点**：
- ✅ 不修改 `.env` 文件
- ✅ 不影响本地开发
- ✅ 一次性操作，安全

### 方法 2: 使用 export（适合多次操作）

```bash
# 1. 设置环境变量（当前终端会话有效）
export DATABASE_URL="postgresql://线上数据库URL"

# 2. 诊断
npm run diagnose:search

# 3. 修复
npm run fix:prod

# 4. 验证
npm run diagnose:search

# 5. 恢复本地配置（可选）
unset DATABASE_URL
```

## 📋 完整操作步骤

### 步骤 1: 获取线上数据库 URL

#### 从 Vercel 获取
1. 登录 Vercel Dashboard
2. 进入你的项目
3. Settings → Environment Variables
4. 找到 `DATABASE_URL`
5. 复制完整的 URL

#### 从 Supabase 获取
1. 登录 Supabase Dashboard
2. 进入你的项目
3. Settings → Database
4. Connection string → URI
5. 复制 URL（记得替换密码）

### 步骤 2: 诊断线上数据库

```bash
# 使用线上数据库 URL 运行诊断
DATABASE_URL="postgresql://..." npm run diagnose:search
```

### 步骤 3: 修复线上数据库

## 🔐 安全建议

### ✅ 推荐做法
```bash
# 使用临时环境变量，不修改文件
DATABASE_URL="postgresql://..." npm run fix:prod
```

### ❌ 不推荐做法
```bash
# 不要直接修改 .env 文件为线上数据库
# 这样容易误操作，影响本地开发
```

### 🛡️ 保护数据库密码
```bash
# 如果需要在脚本中使用，创建临时文件
echo 'DATABASE_URL="postgresql://..."' > .env.production

# 使用时加载
source .env.production
npm run fix:prod

# 用完删除
rm .env.production
```

## 🔍 故障排查

### 问题 1: 连接失败
```
❌ Error: connect ECONNREFUSED
```

**解决**：
- 检查 `DATABASE_URL` 是否正确
- 检查网络连接
- 检查数据库是否在线

### 问题 2: 权限不足
```
❌ Error: permission denied
```

**解决**：
- 检查数据库用户权限
- 确保用户有 UPDATE 权限

### 问题 3: 修复后仍无结果
```
搜索 "数学" 的结果数: 0
```

**检查**：
1. 文章内容是否包含搜索词
2. 文章状态是否为 `published`
3. 文章是否被标记为删除
