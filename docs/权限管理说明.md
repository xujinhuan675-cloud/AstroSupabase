# 权限管理说明

## 概述

系统使用基于角色的访问控制（RBAC），通过 `user_metadata.role` 字段区分用户权限。

## 角色类型

### 1. 管理员 (admin)
- **权限**：可以访问控制台 (`/dashboard`)
- **标识**：`user_metadata.role = 'admin'`
- **用户列表**：
  - User ID: `372a716c-2c7d-44ca-a7fb-3dd221cedb60`
  - User ID: `851af13c-6923-47aa-b25a-95044d032b07`

### 2. 普通用户 (user)
- **权限**：只能访问前台页面
- **标识**：`user_metadata.role = 'user'` 或未设置

## 如何设置管理员

### 方法 1：通过 SQL（推荐）

1. 登录 Supabase Dashboard
2. 进入 SQL Editor
3. 执行 `scripts/set-admin-users.sql` 中的 SQL 语句

### 方法 2：通过代码

管理员白名单已硬编码在：
```typescript
// src/pages/api/auth/wechat/callback.ts
const ADMIN_USER_IDS = [
  '372a716c-2c7d-44ca-a7fb-3dd221cedb60',
  '851af13c-6923-47aa-b25a-95044d032b07'
];
```

新增管理员：
1. 获取用户的 UUID
2. 添加到 `ADMIN_USER_IDS` 数组
3. 重新部署应用

## 前端权限检查

在 `WechatLoginButton.tsx` 组件中：

```typescript
const userRole = user.user_metadata?.role || 'user';
const isAdmin = userRole === 'admin';

// 只有管理员能看到控制台选项
{isAdmin && (
  <a href="/dashboard" className="dropdown-item">
    控制台
  </a>
)}
```

## 数据库结构

用户信息存储在 `auth.users` 表：

```json
{
  "id": "uuid",
  "email": "xxx@wechat.placeholder",
  "raw_user_meta_data": {
    "nickname": "微信昵称",
    "avatar": "头像URL",
    "wechat_openid": "openid",
    "wechat_unionid": "unionid",
    "provider": "wechat",
    "role": "admin" // 或 "user"
  }
}
```

## 查询管理员列表

```sql
SELECT 
  id,
  email,
  raw_user_meta_data->>'nickname' as nickname,
  raw_user_meta_data->>'role' as role,
  created_at
FROM auth.users
WHERE raw_user_meta_data->>'role' = 'admin';
```

## 注意事项

1. **安全性**：管理员 ID 硬编码在代码中，确保代码仓库的访问权限
2. **扩展性**：未来可以创建 `profiles` 表或 `admins` 表来管理权限
3. **验证**：建议在所有需要权限的后端 API 中添加权限检查
4. **日志**：建议记录管理员操作日志
