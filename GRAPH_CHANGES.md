# 图谱组件交互优化 - 修改说明

## 修改内容

### 1. 默认状态
- **节点**：半透明深灰色圆点（alpha: 0.2-0.6），带闪烁动画效果
- **连线**：完全隐藏（alpha: 0）
- **标签**：完全隐藏（alpha: 0）

### 2. 悬浮状态
- **悬浮节点**：实心紫色圆点（#a855f7）
- **相关节点**：空心紫色边框（stroke: 2px, #a855f7）
- **无关节点**：保持闪烁的默认状态
- **连线**：只显示与悬浮节点相关的连线（紫色）
- **标签**：只显示悬浮节点的标签

### 3. 闪烁动画
- 使用正弦波产生平滑的透明度变化
- 透明度范围：0.2 - 0.6
- 每个节点有不同的相位偏移，避免同步闪烁
- 闪烁速度：0.002（约 3 秒一个周期）

## 技术实现

### 修改的文件
- `AstroSupabase/src/components/quartz/scripts/graph-inline.ts`

### 关键修改点

1. **节点初始化**（第 ~600 行）
   - 默认绘制半透明灰色圆点
   - 初始 alpha 设为 0.3

2. **连线初始化**（第 ~650 行）
   - 默认 alpha 设为 0（隐藏）

3. **renderNodes() 函数**（第 ~450 行）
   - 根据悬浮状态动态绘制节点
   - 悬浮节点：实心紫色
   - 相关节点：空心紫色边框
   - 默认节点：半透明灰色

4. **renderLinks() 函数**（第 ~420 行）
   - 只在悬浮时显示相关连线
   - 连线颜色改为紫色（#a855f7）

5. **renderLabels() 函数**（第 ~470 行）
   - 只显示悬浮节点的标签
   - 其他标签完全隐藏

6. **animate() 函数**（第 ~750 行）
   - 添加闪烁动画逻辑
   - 使用正弦波计算透明度
   - 每个节点有独立的相位偏移

## 测试建议

1. 访问首页，观察图谱默认状态
   - 应该看到若隐若现的闪烁点
   - 没有连线和标签

2. 鼠标悬浮在某个节点上
   - 该节点变为实心紫色
   - 显示节点标签
   - 相关节点显示空心紫色边框
   - 显示连接线

3. 移开鼠标
   - 所有节点恢复闪烁状态
   - 标签和连线消失

## 参数调整

如需调整闪烁效果，可修改以下常量（在 animate() 函数前）：

```typescript
const TWINKLE_SPEED = 0.002; // 闪烁速度（越大越快）
const TWINKLE_MIN_ALPHA = 0.2; // 最小透明度
const TWINKLE_MAX_ALPHA = 0.6; // 最大透明度
```

如需调整紫色色值，搜索 `0xa855f7` 并替换为其他颜色的十六进制值。
