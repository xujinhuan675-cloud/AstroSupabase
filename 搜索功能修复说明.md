# 搜索功能修复说明

## 问题现象
线上搜索接口 `/api/search` 始终返回空数组 `[]`，本地搜索正常。

## 根本原因
文章的 `search_vector` 字段为 NULL，导致全文搜索无法匹配任何结果。

可能的原因：
1. 数据库迁移未在线上执行
2. 触发器未创建或未生效
3. 文章是在触发器创建之前导入的

## 快速修复（推荐）

cd AstroSupabase
npm run post-deploy

## 预防措施

### 1. 确保迁移已执行
检查 `drizzle/0003_add_links_support.sql` 是否已在线上数据库执行：

```sql
-- 检查触发器
SELECT trigger_name 
FROM information_schema.triggers
WHERE event_object_table = 'articles'
AND trigger_name = 'articles_search_update';
```

### 2. 新文章自动生成搜索向量
触发器会在插入或更新文章时自动生成 `search_vector`，无需手动操作。

### 3. 定期检查
建议在每次大量导入文章后运行：
```bash
npm run diagnose:search
```

## 技术细节

### 搜索向量生成逻辑
```sql
search_vector = 
  setweight(to_tsvector('simple', coalesce(title,'')), 'A') || 
  setweight(to_tsvector('simple', coalesce(content,'')), 'B')
```

- 使用 `simple` 配置（不分词，适合中英文混合）
- 标题权重 'A'（高优先级）
- 内容权重 'B'（普通优先级）

### 搜索查询逻辑
```typescript
sql`${articles.searchVector} @@ plainto_tsquery('simple', ${searchTerm})`
```

- 使用 `plainto_tsquery` 自动处理用户输入
- 多个词默认为 AND 关系
- 安全防止 SQL 注入

### 性能优化
- GIN 索引: `CREATE INDEX articles_search_idx ON articles USING gin(search_vector)`
- 结果限制: 最多返回 10 条
- 缓存: 搜索结果缓存 60 秒

## 常见问题

### Q: 为什么本地正常但线上不正常？
A: 可能原因：
1. 线上数据库未执行迁移
2. 线上文章是旧数据，没有搜索向量
3. 环境变量配置不同

**解决**: 运行 `npm run post-deploy`

### Q: 修复后搜索还是无结果？
A: 检查：
1. 搜索词是否与文章内容匹配
2. 文章状态是否为 `published`
3. 文章是否被标记为删除 (`is_deleted = true`)

**调试**: 运行 `npm run diagnose:search` 查看详细信息

### Q: 如何支持更好的中文搜索？
A: 当前使用 `simple` 配置，不进行分词。如需更好的中文支持：
1. 安装 `zhparser` PostgreSQL 扩展
2. 修改配置为 `chinese`
3. 或考虑使用 Elasticsearch

### Q: 搜索性能如何？
A: 当前性能：
- 使用 GIN 索引，查询速度快
- 限制返回 10 条结果
- 缓存 60 秒

对于大量数据，可以考虑：
- 增加缓存时间
- 使用 Redis 缓存
- 分页加载
