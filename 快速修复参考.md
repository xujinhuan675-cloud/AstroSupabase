# 搜索功能快速修复参考

## 🎯 核心概念

**所有脚本修复的是 `DATABASE_URL` 指向的数据库**

```
.env 文件中的 DATABASE_URL → 决定修复哪个数据库
```

---

## 📍 修复本地数据库

```bash
# 确保 .env 配置的是本地数据库
# DATABASE_URL=postgresql://localhost:5432/local_db

# 诊断
npm run diagnose:search

# 修复
npm run fix:search
```

---

## 🌐 修复线上数据库

### 推荐方法：临时环境变量

```bash
# 1. 从 Vercel 获取线上数据库 URL
# Settings → Environment Variables → DATABASE_URL

# 2. 一次性修复（不修改 .env）
DATABASE_URL="postgresql://user:pass@host:5432/db" npm run fix:prod
```

### 详细步骤

```bash
# 步骤 1: 诊断线上数据库
DATABASE_URL="postgresql://..." npm run diagnose:search

# 步骤 2: 修复线上数据库
DATABASE_URL="postgresql://..." npm run fix:prod

# 步骤 3: 测试线上搜索 API
curl "https://your-domain.vercel.app/api/search?q=数学"
```

---

## 🚀 部署流程

```bash
# 1. 部署到 Vercel
npm run deploy

# 2. 部署后修复线上数据库
DATABASE_URL="postgresql://..." npm run fix:prod

# 3. 验证搜索功能
curl "https://your-domain.vercel.app/api/search?q=测试"
```

---

## 📋 可用命令

| 命令 | 用途 | 修复对象 |
|------|------|---------|
| `npm run diagnose:search` | 诊断搜索功能 | DATABASE_URL 指向的数据库 |
| `npm run fix:search` | 修复搜索索引 | DATABASE_URL 指向的数据库 |
| `npm run fix:prod` | 自动检查并修复 | DATABASE_URL 指向的数据库 |
| `npm run post-deploy` | 同 fix:prod | DATABASE_URL 指向的数据库 |

## ⚠️ 重要提示

### ✅ 推荐做法
```bash
# 使用临时环境变量，安全不影响本地
DATABASE_URL="postgresql://..." npm run fix:prod
```

### ❌ 不推荐做法
```bash
# 不要修改 .env 文件为线上数据库
# 容易误操作，影响本地开发
```

## 🆘 常见问题

### Q: 如何获取线上数据库 URL？
**A**: Vercel Dashboard → 项目 → Settings → Environment Variables → DATABASE_URL

### Q: 修复后还是返回空数组？
**A**: 
1. 运行诊断确认 search_vector 已生成
2. 检查搜索词是否与文章内容匹配
3. 确认文章状态为 published

### Q: 可以在 Vercel 上自动修复吗？
**A**: 不推荐。因为：
- 需要数据库连接权限
- 可能影响部署时间
- 手动修复更可控
